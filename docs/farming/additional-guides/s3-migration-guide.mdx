# Migrating from AWS S3 to Autonomys Auto Drive

Autonomys Auto Drive features an AWS S3-compatible API layer. If your existing application utilizes standard object storage, you can migrate your backend data pipelines to permanent, decentralized storage by swapping your client configuration endpoint and access credentials—without rewriting your core application logic.

### Prerequisites

- An active Auto Drive account with API credentials (Access Key and Secret Key).
- The standard AWS SDK installed in your development environment.

## Node.js Implementation Example

Below is a minimal configuration demonstrating how to overwrite the default AWS S3 client endpoint to target the Auto Drive S3-fluent architecture using the official `@aws-sdk/client-s3` library.

```javascript
const { S3Client, PutObjectCommand } = require("@aws-sdk/client-s3");

// Initialize the S3 Client pointed at the Auto Drive Endpoint
const autoDriveClient = new S3Client({
  region: "us-east-1", // Placeholder region required by the SDK
  endpoint:
    "[https://s3.auto-drive.autonomys.xyz](https://s3.auto-drive.autonomys.xyz)", // Replace with official Auto Drive S3 gateway URL
  credentials: {
    accessKeyId: process.env.AUTONOMYS_ACCESS_KEY_ID,
    secretAccessKey: process.env.AUTONOMYS_SECRET_ACCESS_KEY,
  },
  forcePathStyle: true, // Enforces path-style addressing for object resolution
});

async function uploadToPermanentStorage(bucketName, fileKey, fileBuffer) {
  const command = new PutObjectCommand({
    Bucket: bucketName,
    Key: fileKey,
    Body: fileBuffer,
  });

  try {
    const response = await autoDriveClient.send(command);
    console.log("🔒 Data written permanently to Autonomys!");
    console.log("Decentralized ETag (CID Mapping):", response.ETag);
    return response;
  } catch (error) {
    console.error("Migration Upload Error:", error);
    throw error;
  }
}
```
